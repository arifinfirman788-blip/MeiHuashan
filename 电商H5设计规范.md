# 六盘水梅花山景区商品详情 H5 设计规范

## 1. 设计定位

梅花山不是城市历史街区，页面不再使用文昌阁式的城市文化叙事。整体定位为：

> **云上森林里的景区票务页**

视觉要让游客感受到高山、森林、清凉和度假感，同时保持传统景区购票页面的清晰与可信。

页面首先回答：

```text
这是梅花山的什么票 → 适合谁 → 多少钱 → 哪天能用 → 如何入园
```

## 2. 设计关键词

- 高山：大面积留白、远山视野、轻盈的上下层次；
- 森林：深松绿、雾白、低饱和自然色；
- 梅花：只用一处梅花粉表达当前选择和景区识别；
- 度假：信息不拥挤，价格和预订按钮始终明确；
- 票务：票型直接出现在详情页，不把核心选择藏进弹层。

## 6. 页面结构


整体布局参考附件的视觉骨架：大圆角主图、三项指标、胶囊分类切换、单列内容和底部行动栏；不复用附件中的英文文案和业务内容。

## 7. 票型设计规则

票型是商品详情页的核心选择，放在独立的“票型价格”Tab 中，同时在底部购买栏持续显示当前选择。

- 默认展示第一个可售票型，票型通过数量加减选择，不使用整卡选中状态；
- 每个票型显示名称、适用人群、价格和库存状态；
- 售罄票型保留展示，但不可点击；
- 票型选择后，底部价格、当前票型、票型标签和保险说明同步变化；
- 库存状态直接放在票型卡内，不单独展示后台库存类型；
- 每种票型独立配置购买数量，支持成人票、儿童票等多种票型同时购买；
- 点击“立即预订”后选择游玩日期、入园时段并填写联系人信息和实名游客信息；
- 联系人姓名、联系人手机号必填；游客姓名、证件号必填，游客手机号选填；
- 订单汇总按票型和数量展示，提交前校验联系人和游客信息完整性；
- 提交订单后调用微信支付，支付完成后进入支付结果页；
- 提交订单成功后使用服务端返回的 `orderNo`、`payAmount` 调用 `wechatPayment`，将 `payInfo` 传入 `wx.requestPayment`；
- `wx.requestPayment` 成功后仍调用 `getOrderByNo` 查询订单状态，不能仅凭前端按钮点击或客户端支付回调判断出票成功；
- 支付成功但订单仍在出票时，显示“支付成功，正在出票”，出票完成后再展示核销码；
- `resourcesSkuList` 对应详情页票型列表；
- `skuPriceList[].price` 对应票型价格；
- `hasRealName` 和 `checkType` 控制游客信息表单。

## 9. 状态规范

- 商品加载中；
- 无可售票型；
- 票型售罄；
- 日期售罄；
- 时段售罄；
- 实名信息未填写；
- 创建订单失败；
- 支付中；
- 支付成功；
- 出票中；
- 出票成功；
- 出票失败；
- 退款中；
- 已退款。

支付成功但票码尚未生成时，显示“已支付成功，正在出票，请稍后查看订单”，不能提示支付失败。

### 9.1 支付与订单状态流转

```text
待填写订单 → 创建订单中 → 待支付 → 支付处理中 → 支付成功/出票中 → 出票成功
                         ↘ 支付失败
                         ↘ 用户取消/订单关闭
支付成功/出票中 → 出票失败（进入订单处理）
已支付订单 → 退款中 → 已退款
```

- `createOrder` 失败：停留在预订弹层，不进入支付结果页；
- `wechatPayment` 获取参数失败：展示支付未发起，允许重试，并查询订单状态；
- 用户取消 `wx.requestPayment`：保留订单号，不展示支付成功，订单仍待支付时允许再次支付；
- 微信支付成功：调用 `getOrderByNo`，根据最新 `orderStatus` 展示支付成功、出票中或出票成功；
- `orderStatus` 的具体枚举值、出票完成标识和状态切换时机：待黔云联创确认；
- 只有订单确认已出票后，才调用核销码接口展示 `voucherCodes`。

当前样稿中的山地风景图只是梅花山风格占位图，正式上线应替换为六盘水梅花山景区官方实拍素材，优先使用：

- 山地远景；
- 云雾森林；
- 索道或山顶视角；
- 冬季雪景或梅花季素材；
- 能体现高山度假感的游客视角。
